# Contributing

`jschwefel-workshop` is a home for standalone, one-off tools — each repo is independent and may have
its own conventions, stack, and license. There is no shared framework to learn.

## How

1. **Open an issue first** for anything non-trivial — describe the problem, not just a patch.
2. **Small, focused PRs.** One concern each.
3. **Check the repo's own README** for anything specific to that tool.

Be respectful (see the Code of Conduct). Contributions are under each repository's stated license.
